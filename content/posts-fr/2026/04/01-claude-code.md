# Claude Code

Aujourd'hui, le code source de l'application Claude Code a été fuité (ou divulgué). C'est une grande nouvelle.

Mais je crois que la plupart des gens se trompent : ils tirent la mauvaise conclusion.

Certaines personnes croient que la qualité du code source est très mauvaise. Il y a des fonctions de plus de deux mille lignes, du code répété, des bugs logiciels, etc. Je crois qu'il y a un message important ici : à ce stade, la qualité du code n'a pas d'importance.

Si la qualité du code déterminait la qualité du produit, alors tout logiciel qui change le monde aurait du code de haute qualité. Or, ce n'est pas le cas. Donc, la qualité du code ne détermine pas la qualité du produit.

Je crois que ce qui importe désormais n'est pas la performance des ingénieurs humains, mais le taux de défauts d'un nouveau cycle de développement et de livraison. Ce nouveau cycle s'insère dans la phase de développement du cycle traditionnel. Le taux d'erreur effectif est tout ce qui compte.

La vérification formelle et l'automatisation de la conformité constitueront le prochain point d'appui.
