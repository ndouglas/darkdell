Arbor is a Rust library for procedurally generating biologically diverse non-player character species. The library provides users with a way to create a wide range of species, from goblins and elves to fantastical creatures like dragons and griffins. It also includes tools for generating the anatomical features of these species, such as body types, sensory organs, and other physical attributes.

The library is built using Rust and incorporates a range of algorithms and techniques from fields such as biology, anatomy, and natural selection. It is designed to be highly customizable and extendable, so that users can easily add their own algorithms and features to the system.
<!--more-->

### Naming

Arbor is the name of the ancestral seat of House Redwyne in the A Song of Ice and Fire series, and is also the Latin word for "tree". This name is fitting for a library that aims to create biologically diverse non-player characters, and also alludes to the library's use of evolutionary algorithms to simulate the process of natural selection.

## Motivations

The motivation for this project comes from a deep interest in biology and the diversity of life on Earth, as well as a fascination with the idea of creating complex and realistic virtual worlds. The goal of Arbor is to provide a tool that makes it easier for other developers to explore these topics and create their own virtual worlds that are scientifically accurate and richly detailed.

## Challenges

One of the main challenges of this project is the complexity of the biological and anatomical systems involved. Creating a system that can generate realistic and diverse non-player characters, with all the necessary physical and sensory characteristics, is a daunting task that requires a deep understanding of biology and anatomy.

To tackle this challenge, we plan to start with the basics and build up from there, gradually adding more complex features and refining the algorithms as we go. We will also be consulting with experts in the fields of biology and anatomy to ensure that our models and algorithms are as accurate and up-to-date as possible.
