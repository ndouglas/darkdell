The library is designed to be highly customizable, allowing game developers to adjust the parameters of the combat engine to suit their specific needs.

## Summary

Sharp Point is a Rust library for building complex combat engines for games. The library provides game developers with the tools they need to create detailed and realistic combat systems that take into account various factors, such as the anatomy of the combatants, the weapons or fighting styles they use, and any magical elements that may be involved.

The library is built using Rust and incorporates a range of algorithms and techniques from fields such as anatomy, physics, and game design. It is designed to be highly customizable and extendable, so that game developers can easily add their own algorithms and features to the system.

### Naming

Sharp Point is a castle located on the coast of the Stormlands, and its name suggests the idea of precision, accuracy, and deadly force. These are all elements that are important in a combat engine for games, making "Sharp Point" an appropriate name for your project.

## Motivations

The motivation for this project comes from a desire to create games with combat systems that are more detailed, realistic, and engaging. Traditional combat engines often overlook many of the nuances of real-world combat, such as the impact of weapon weight on attack speed or the effects of different fighting styles on stamina. By building a library that takes these factors into account, we hope to provide game developers with a tool that will help them create more immersive and satisfying combat experiences.

## Challenges

One of the main challenges of this project is the complexity of the task at hand. Creating a combat engine that is both detailed and realistic requires a deep understanding of anatomy, physics, and game design, as well as a willingness to experiment and try new approaches.

To tackle this challenge, we plan to start with the basics and build up from there, gradually adding more complex features and refining the algorithms as we go. We will also be consulting with experts in the fields of anatomy, physics, and game design to ensure that our models and algorithms are as accurate and up-to-date as possible.
