## Summary

Stokeworth is a domain-specific programming language that is designed to provide a high-level scripting interface and an artificial intelligence system for roguelike game developers. The language is built on top of Rust and is designed to be expressive, easy-to-use, and extensible.

The language includes a just-in-time (JIT) compiler that is implemented using Rust and follows Robert Nystrom's _Crafting Interpreters_ book. This compiler enables fast and efficient execution of Stokeworth scripts and provides game developers with a powerful tool for controlling game entities and implementing complex artificial intelligence systems.

### Naming

Stokeworth is the name of a small castle in the Crownlands region of Westeros. The name suggests the idea of igniting or kindling a fire, which could be a fitting metaphor for a programming language that is designed to bring entities to life within a game world.

## Motivations

The motivation for this project comes from a desire to create a simple and expressive programming language that is specifically designed for roguelike game development. The goal of Stokeworth is to make it easier for me to create sophisticated and engaging game worlds, while also enabling them to quickly and easily implement complex artificial intelligence systems.

## Challenges

One of the main challenges of this project is to design a language that is both expressive and easy-to-use, while also being powerful enough to handle the complex demands of game development. To address this challenge, I plan to focus on a clean and expressive syntax and provide a rich set of built-in features and libraries that are specifically designed for roguelike game development.
